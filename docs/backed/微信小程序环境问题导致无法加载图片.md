# 报错

VM75:363 Error: not node js file system!path:/saaa_config.json; go **invokeHandler** readFile worker? false

# 疑似原因

运行时尝试通过 Node.js 的 fs.readFile 去读取 /saaa_config.json，但当前环境是 H5/小程序 WebView，并不存在 Node 文件系统，触发 not node js file system! 异常；因为配置未加载成功，依赖该配置的静态资源路径重写没有生效，最终图片请求用到了错误的地址而无法展示。
项目目录内（包含 static、uni_modules、unpackage 等）未找到 saaa_config.json，说明对应插件或自动注入的资源映射配置缺失；代码中也没有显式的 Node fs 调用，推断为第三方或构建期注入逻辑在运行时执行。
读取配置 → 静态资源路径重写（或 CDN/OSS 基址计算） → <image> 真正请求地址；配置读失败让这条链路中断，因此图片源变成空串或错误相对路径。

- 小程序启动时，某个模块（可能是 safe-area-insets 或调用它的组件）执行了初始化逻辑
- 该逻辑尝试用 readFile 读取 /saaa_config.json
- 微信环境识别出这是非法操作，抛出错误

# 排查

1. 项目中确实不存在saaa_config.json
2. 在根目录创建空的saaa_config.json无法解决问题
3. 卸载了safe-area-insets包，无效果
