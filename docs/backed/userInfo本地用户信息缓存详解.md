# 生命周期

┌─────────────────────────────────────────────────────┐
│ 数据库（后端） │
│ 用户表：{ user_id, user_name, gender, avatar... } │
└─────────────────────────────────────────────────────┘
↑ 保存 ↓ 获取
│ │
┌─────────────────────────────────────────────────────┐
│ 后端 API 接口 │
│ /user/update - 更新用户信息 │
│ /user/login - 登录时返回用户信息 │
│ /user/info - 获取用户信息（可选） │
└─────────────────────────────────────────────────────┘
↑ ↓
│ │
┌─────────────────────────────────────────────────────┐
│ 前端本地存储（userInfo） │
│ { user_name, gender, access_token, ... } │
│ 作用：缓存，避免频繁请求接口 │
└─────────────────────────────────────────────────────┘

# 设计意义

1. 本地缓存可以减少网络请求，某些请求直接读取本地缓存，提升性能，减少服务器压力
2. 网络断开后，通过本地缓存，依然可以显示基础信息

# 数据同步策略

数据库是真实数据源，本地存储是缓存副本
登录时会从数据库同步到本地，对数据库修改也会对本地同步更新
